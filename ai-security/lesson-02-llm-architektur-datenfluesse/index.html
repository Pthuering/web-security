Modul 2: LLM-Architektur & Datenflüsse

Prompt → Model → Output, RAG, Tool Use, Agent Loops — wo fließen Daten, wo entstehen Risiken?

Schwierigkeitsgrad
Zeitschätzung
~35 Minuten
Cert-Mapping
SecAI+ Domain 1 (17%)

📋 Voraussetzungen

Modul 1: AI/ML-Grundlagen für Security — Du brauchst die ML-Taxonomie, Training vs. Inference Pipeline, und die Attack Surface Map als Basis.

🎯 Lernziele

  • Die fünf LLM-Architekturmuster (Direct Chat, RAG, Tool Use, Agent Loop, Multi-Model) unterscheiden können
  • Datenflüsse durch jede Architektur nachzeichnen und Trust Boundaries identifizieren
  • Tool Use / Function Calling als Security-relevantes Konzept verstehen
  • Agent Loops und deren Eskalationspotenzial erklären können
  • Für eine gegebene LLM-Anwendung das Architekturmuster und den primären Risikovektor benennen können
