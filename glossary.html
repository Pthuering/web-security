Glossar

Alle Kernbegriffe aus beiden Tracks — wächst mit jeder Lektion.

  • Cross-Site Scripting (XSS)
    Eine Klasse von Sicherheitslücken, bei der ein Angreifer eigenen JavaScript-Code im Browser eines anderen Nutzers ausführen lässt. Drei Varianten: Reflected, Stored, DOM-based.
    → Web-Security Lesson 01
  • HTML Injection
    Das Einschleusen von HTML-Markup in eine Webseite über nicht-sanitisierte Benutzereingaben. Oft Vorstufe oder Bestandteil von XSS.
    → Web-Security Lesson 01
  • Input Sanitization
    Das Bereinigen von Benutzereingaben, bevor sie verarbeitet oder angezeigt werden — z.B. durch Entfernen oder Escapen von HTML-Sonderzeichen.
    → Web-Security Lesson 01
  • Output Encoding
    Das kontextgerechte Umwandeln von Sonderzeichen (z.B. < → &lt;), bevor Daten in HTML, JavaScript oder andere Ausgabekontexte geschrieben werden.
    → Web-Security Lesson 01
  • Payload
    Der ausführbare Teil eines Angriffs — bei XSS typischerweise ein JavaScript-Snippet wie <script>alert(1)</script>.
    → Web-Security Lesson 01
  • Reflected (Non-Persistent)
    Eine XSS-Variante, bei der die bösartige Eingabe vom Server sofort in die Antwort reflektiert wird, ohne gespeichert zu werden. Der Angriff steckt typischerweise in der URL.
    → Web-Security Lesson 01
  • URL-Parameter (Query String)
    Key-Value-Paare nach dem ? in einer URL (z.B. ?search=hello). Häufiger Eintrittspunkt für Reflected XSS, da sie vom Nutzer kontrolliert werden.
    → Web-Security Lesson 01
